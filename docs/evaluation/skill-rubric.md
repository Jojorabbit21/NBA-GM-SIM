# 능력치 절대 평가 기준 — 플레이 정의 루브릭 (Skill Rubric)

## 개요

### 이 문서의 목적
선수 능력치를 **스탯이 아닌 기술 자체**로 정의한다. 스탯 기반 평가의 4가지 구조적 결함을 해결하기 위해 작성되었다:
1. 스탯이 없는 신인도 영상/관찰로 평가 가능
2. 능력치 = 원인 (스탯 = 결과물이므로 1차 기준 사용 불가)
3. 평가자 주관 최소화 — 명확한 플레이 묘사로 일관성 확보
4. 시대 무관 — 기술 자체를 평가하므로 시대 보정 불필요

### 3계층 루브릭 구조

```
계층 1: 게임 엔진 결과
  "이 능력치가 X이면 PBP 엔진에서 Y% 확률로 이 결과가 나온다"
  → 순수 수학적, 완전 객관적

계층 2: 플레이 묘사 (평가의 1차 기준)
  "이 수치의 선수는 이런 상황에서 이런 플레이를 한다"
  → 영상/관찰만으로 평가 가능 — 스탯 불필요

계층 3: 스탯 참고값 (교차 검증 도구)
  "일반적으로 이 수준 선수는 이런 스탯 범위를 보인다"
  → 전술/팀에 의해 왜곡된 경우 무시
```

### 수치 구간 정의
| 구간 | 명칭 | 의미 |
|------|------|------|
| 95-99 | 역대급 | NBA 역사 해당 스킬 Top 3 이내 |
| 88-94 | 엘리트 | 현역 상위 1~3% |
| 78-87 | 우수 선발 | 상위 10~15%, 올스타급 |
| 65-77 | 리그 평균 | 주전 수준 |
| 50-64 | 로테이션 | 벤치 상위, 특정 상황 제한 |
| 20-49 | 심각한 약점 | 해당 기술 극히 부족 |

# 부록: 평가 절차

## 신규 선수 평가 순서

1. **신체 능력치 먼저** (speed, agility, vertical, strength)
   - Combine 측정값 있으면 직접 변환
   - 없으면 영상에서 리그 평균 선수와 비교 관찰

2. **완전 객관적 능력치** (ft, layup, dunk, mid, three*, stl, blk, drawFoul)
   - NBA 스탯 있으면: 스탯 참고값 확인 → 플레이 묘사와 대조
   - 스탯 없으면: 플레이 묘사만으로 평가 (스탯 없어도 가능)

3. **복합 능력치** (handling, passing 3종, shotIq, offConsist, defConsist)
   - 플레이 묘사를 1차 기준으로 평가
   - 스탯은 교차 검증용

4. **무형 능력치** (intangibles, helpDefIq, passPerc)
   - 플레이 묘사만으로 판단
   - 대형 경기/플레이오프 이력 참고

## 이의 제기 원칙

특정 선수의 능력치에 이의가 있을 경우:
1. 루브릭의 **플레이 묘사**와 실제 플레이 비교
2. 묘사와 불일치하면 → **루브릭 적용 오류** → 능력치 조정
3. 묘사와 일치하는데 스탯이 다르면 → **스탯이 전술/팀에 의해 왜곡** → 능력치 유지
4. 루브릭 자체가 잘못됐다면 → **루브릭 개정** (선수 능력치 수정 전에)
