# 선수 등급 분류 시스템 (Player Grades)

## 1. 개요

### 목적
능력치 재평가의 기준점 확립을 위해 전체 선수를 **S+~F 12단계 등급**으로 분류한다.

### OVR vs Grade 차이
| 구분 | OVR | Grade |
|------|-----|-------|
| 기반 | 아키타입·희귀조합 보너스 포함 | 순수 원시 능력치만 |
| 용도 | 트레이드 밸류, 팀 전력 계산 | 능력치 재평가 기준, 스카우팅 |
| 포지션 보정 | 없음 (절대값) | 포지션별 가중치 적용 |

### Position Score 계산 방식
각 포지션별 핵심 능력치의 **가중합**으로 0~100 사이의 점수를 계산한다.
```
Position Score = Σ(핵심능력치_i × 가중치_i)
```
- 성장/퇴화(`attrDeltas`)가 적용된 현재 능력치로 계산 → 자동 반영
- 런타임에 매번 계산 가능 (경량 연산, 캐싱 불필요)

## 2. 공통 등급 정의 (12단계)

| 등급 | NBA 수준 | 설명 |
|------|---------|------|
| **S+** | 역대 최정상 / 시대 정의 | 올-NBA 1팀 반복, HOF 확정급. 포지션 역사상 2~3명 |
| **S** | 슈퍼스타 | 프랜차이즈 코너스톤, MVP/FMVP 수상권. 팀 빌딩의 핵심 자산 |
| **A+** | 올스타 핵심 | 올스타 선정, 포지션 상위 6~10%. 트레이드 핵심 자산 |
| **A** | 올스타급 주전 | 컨텐더 팀의 2번째 옵션. 안정적 30분+ 주전 |
| **A-** | 우수 주전 | 플레이오프 주전 수준. 컨텐더의 3번째 옵션 |
| **B+** | 평균 이상 주전 | 리그 평균 이상 주전. 중위권 팀의 2번째 옵션 |
| **B** | 리그 평균 주전 | 30개 팀 로스터에서 안정적 주전 선수 |
| **B-** | 벤치 상위 | 로테이션 벤치, 특정 상황 전문가 (3&D, 에너지맨) |
| **C** | 로테이션 벤치 | 특정 역할 제한적. 정규 로테이션 하위 |
| **C-** | 한계 로스터 | 15인 로스터 마지막 자리, 2-way 계약 수준 |
| **D** | 리그 마지노선 | 10-day 계약, 주기적 컷 대상 |
| **F** | 리그 미달 | G리그 수준, 현 시점 NBA 경쟁력 없음 |

## 8. 보정 규칙

### 8-1. 플레이메이킹 빅 보정 (Playmaking Big Boost)
요키치·마크 가솔처럼 C/PF 포지션에서 플레이메이킹이 극도로 높은 선수에게 추가 보정.

```
조건: (C 또는 PF) AND piq ≥ 88 AND pvis ≥ 85
보정: Score += (piq - 65) × 0.04 + (pvis - 65) × 0.03
```

적용 예:
- 요키치 (piq=98, pvis=99): +1.32 + 1.02 = **+2.34점**
- 도만타스 사보니스 (piq=92, pvis=88): +1.08 + 0.69 = **+1.77점**

### 8-2. 수비 스페셜리스트 최저 등급 보장

```
C: idef ≥ 90 AND blk ≥ 88 → 최소 B- (Score ≥ 68.0) 보장
PG/SG: pdef ≥ 90 AND stl ≥ 85 → 최소 C+ (Score ≥ 62.0) 보장
```

### 8-3. 멀티포지션 선수 처리
등록 포지션(meta_players.position) 기준으로 Grade 산출.
- 루카 돈치치 → PG Score 기준
- 르브론 제임스 → SF Score 기준 (SF 등록 시)

## 9. Score 계산 SQL 참조

### PG Score (Supabase 쿼리)
```sql
SELECT
  name,
  ROUND(
    (base_attributes->>'handl')::numeric      * 0.22 +
    (base_attributes->>'piq')::numeric        * 0.15 +
    (base_attributes->>'pvis')::numeric       * 0.13 +
    (((base_attributes->>'3c')::numeric
      + (base_attributes->>'3_45')::numeric
      + (base_attributes->>'3t')::numeric) / 3) * 0.12 +
    (base_attributes->>'siq')::numeric        * 0.10 +
    (base_attributes->>'pdef')::numeric       * 0.10 +
    (base_attributes->>'stl')::numeric        * 0.08 +
    (base_attributes->>'intangibles')::numeric * 0.06 +
    (base_attributes->>'spd')::numeric        * 0.04
  , 1) AS pg_score
FROM meta_players
WHERE position = 'PG' AND (draft_year IS NULL OR draft_year != 2026)
ORDER BY pg_score DESC;
```

### C Score (Supabase 쿼리)
```sql
SELECT
  name,
  ROUND(
    (base_attributes->>'idef')::numeric       * 0.22 +
    (base_attributes->>'dreb')::numeric       * 0.16 +
    (base_attributes->>'post')::numeric       * 0.14 +
    (base_attributes->>'blk')::numeric        * 0.12 +
    (base_attributes->>'oreb')::numeric       * 0.10 +
    (base_attributes->>'str')::numeric        * 0.08 +
    (base_attributes->>'intangibles')::numeric * 0.08 +
    (base_attributes->>'piq')::numeric        * 0.06 +
    (base_attributes->>'dnk')::numeric        * 0.04
  , 1) AS c_score
FROM meta_players
WHERE position = 'C' AND (draft_year IS NULL OR draft_year != 2026)
ORDER BY c_score DESC;
```
