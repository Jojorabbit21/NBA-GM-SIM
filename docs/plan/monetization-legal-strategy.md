# NBA GM SIM 수익화 전략 & 법적 리스크 분석

## Context
실명 선수를 사용하는 가상 리그 시뮬레이션의 상업화를 위해, 저작권/퍼블리시티권 리스크를 분석하고 안전한 수익화 경로를 설계한다.

## Part 2: 법률 도메인별 상세 분석

### 2.1 저작권 (Copyright) — 현재 **안전**
- 팀명/로고: 이미 가상으로 교체 완료
- 게임 규칙/메카닉: 아이디어는 저작권 대상 아님 (Football Manager, OOTP 선례)
- 선수 스탯: 사실 데이터는 저작권 보호 불가, 능력치도 독자 생성

### 2.2 퍼블리시티권 — **핵심 리스크**

**한국법 근거:**
- 부정경쟁방지법 제2조 제1항 차목 (2018년 개정): 타인의 성명·초상 등을 무단 상업적 이용 금지
- 성명권 (민법 764조 유추)

**유리한 논거:**
1. 관리/전략 시뮬레이션 — 선수를 스탯으로 표현, 초상/음성 사용 없음
2. 한글 음차는 사실적 식별(factual identification)
3. 가상 팀 맥락이 실제 NBA와 거리를 둠

**불리한 논거:**
1. 유료 상업 제품에서 실명을 사용해 유저를 유인
2. 한국 법원은 퍼블리시티권을 확대 해석하는 추세

**유사 게임 선례:**
| 게임 | 접근법 | 결과 |
|------|--------|------|
| Football Manager | FIFPro 통해 선수 이름 라이선스 구매 | 합법 상업화 |
| OOTP Baseball | MLBPA 라이선스 | 합법 상업화 |
| Basketball GM (dumbmatter) | 완전 무료·오픈소스 유지 | 수익화 안 함 (의도적) |
| EA NCAA Football | 실명 사용 → O'Bannon 소송 패소 | 10년간 시리즈 중단 |

### 2.3 상표 (Trademark) — 현재 **안전**
- NBA 워드마크/로고 사용 없음
- 팀명 가상, 도시명은 지명으로 상표 대상 아님

### 2.4 게임물 등급분류 규제 — **회피 필수**

> **핵심 원칙: 본 서비스는 "게임"이 아닌 "스포츠 시뮬레이션 플랫폼/도구"로 포지셔닝해야 한다.**

#### 2.4.1 왜 "게임"으로 분류되면 안 되는가

**게임산업진흥에 관한 법률 (게임산업법)** 에 의해 "게임물"로 분류되면:

1. **등급분류 의무**: 게임물관리위원회(게관위)에 등급분류 심의 필수 — 비용 수십만~수백만 원, 수주~수개월 소요
2. **사후 규제**: 등급분류 후에도 내용 변경 시 재심의 필요 (라이브 서비스에 치명적)
3. **사행성 심사**: 확률형 아이템, 결과 불확실성이 있으면 사행성 심사 대상
4. **연령등급 표시 의무**: 모든 마케팅·UI에 등급 표시
5. **영업시간 제한**: 과거 셧다운제는 폐지되었으나, 청소년 대상 규제 잔존
6. **과징금/영업정지**: 미등급 게임물 유통 시 2년 이하 징역 또는 2천만 원 이하 벌금 (게임산업법 제44조)

#### 2.4.2 "게임물"의 법적 정의와 회피 논거

**게임산업법 제2조 제1호** — 게임물 정의:
> "컴퓨터프로그램 등 정보처리 기술이나 기계장치를 이용하여 오락에 적합하도록 제작된 **영상물 또는 그 영상물의 이용을 주된 목적**으로 제작된 기기 및 장치"

**게임물 제외 대상 (동법 제2조 제1호 단서):**
- 가. 사행행위에 이용되는 것
- 나. 「관광진흥법」에 의한 관광사업
- 다. **「정보통신망법」에 의한 정보제공 서비스** ← 핵심
- 라. **교육·학습·연구 등의 용도로 제작된 것** ← 활용 가능

**"게임물이 아니다"를 주장하기 위한 핵심 논거:**

| 논거 | 설명 | 우리 서비스 해당 여부 |
|------|------|---------------------|
| **정보 제공 서비스** | 선수 데이터 분석·시뮬레이션 결과를 정보로 제공 | **해당** — 선수 스탯, 시즌 시뮬레이션 결과 등 정보 제공이 핵심 |
| **도구/유틸리티** | 결과를 조작/소비하는 오락이 아닌, 사용자가 의사결정을 테스트하는 도구 | **해당** — GM 의사결정(로스터, 전술)의 결과를 시뮬레이션하는 분석 도구 |
| **오락 주목적 아님** | "오락에 적합하도록 제작"이 아닌 다른 주된 목적 존재 | 포지셔닝에 따라 주장 가능 |
| **영상물 아님** | 실시간 그래픽 렌더링, 캐릭터 조작 등 영상물적 요소 부재 | **해당** — 데이터/텍스트/통계 기반, 3D 그래픽 없음 |

#### 2.4.3 유사 서비스 선례 분석

| 서비스 | 유형 | 게관위 등급분류 | 수익화 |
|--------|------|---------------|--------|
| **판타지 스포츠 (팬즈픽, 스포키 등)** | 실제 선수 기반 포인트 경쟁 | **미분류** — 정보 제공 서비스로 분류 | 참가비/구독 |
| **네이버 스포츠 시뮬레이션** | 경기 결과 예측 | **미분류** — 정보 서비스 | 광고 기반 |
| **Football Manager** | 스포츠 경영 시뮬레이션 | **등급분류 받음** (전체이용가) — 스스로 "게임"으로 포지셔닝 | 패키지 판매 |
| **Yahoo Fantasy Sports** | GM 시뮬레이션 + 실제 경기 연동 | **미분류** (정보 서비스) | 프리미엄 구독 |
| **Basketball Reference** | 선수 스탯 + 시뮬레이션 도구 | **미분류** — 정보 제공 | 광고/구독 |

**핵심 관찰**: 한국의 판타지 스포츠 플랫폼들은 "정보 제공 서비스"로 분류되어 게관위 심의 없이 운영 중. 동일한 포지셔닝이 가능.

#### 2.4.4 "게임 아닌 시뮬레이션 플랫폼" 포지셔닝 전략

**A. 서비스 명칭·브랜딩에서 "게임" 용어 완전 배제**

현재 문제점:
- `APP_NAME = 'Basketball GM Simulator'` — "Simulator"는 괜찮으나 맥락상 게임으로 읽힐 수 있음
- UI/마케팅에서 "게임", "플레이", "승리" 등의 용어 사용 여부 점검 필요

권장 변경:
- 서비스명: **"Basketball GM Lab"** 또는 **"Basketball Front Office"** 등 도구/플랫폼성 명칭
- 서비스 설명: "농구 프런트 오피스 시뮬레이션 플랫폼" / "GM 의사결정 시뮬레이터"
- UI 용어: "플레이" → "시뮬레이션 실행", "게임" → "경기 시뮬레이션", "승리" → "시뮬레이션 결과"

**B. 서비스 구조를 "정보 제공" 중심으로 설계**

게관위가 게임물 여부를 판단할 때 보는 핵심 기준:
1. **오락성이 주된 목적인가?** → "아니오, 분석/시뮬레이션 도구"
2. **이용자가 결과에 직접 관여하는가?** → "GM 파라미터 설정 후 시뮬레이션 엔진이 자동 산출"
3. **경쟁적 승패가 있는가?** → "시뮬레이션 결과 제공이지, 유저 간 승패 경쟁이 아님"

구체적 설계 원칙:
- **시뮬레이션 결과를 "분석 리포트" 형태로 강조**: 박스스코어, 시즌 통계, 트렌드 분석 → 데이터 제공 서비스
- **멀티플레이어도 "리그 시뮬레이션"으로 프레이밍**: 30명이 각자 GM 의사결정 → 시뮬레이터가 결과 산출 → "경영 시뮬레이션 플랫폼"
- **PBP(Play-by-Play)**: "경기 시뮬레이션 로그"로 포지셔닝 (게임 실황이 아닌 분석 데이터)

**C. 이용약관·서비스 분류에서 법적 근거 확보**

`legalTexts.ts`에 추가할 내용:
```
본 서비스는 게임산업진흥에 관한 법률 제2조 제1호 다목 및 라목에 해당하는
정보 제공 서비스 및 연구·분석 용도의 시뮬레이션 플랫폼으로서,
동법상 "게임물"에 해당하지 않습니다.
```

**D. 사업자 등록 시 업종 분류**

- **피해야 할 업종**: "게임 소프트웨어 개발 및 공급업" (KSIC 58210)
- **권장 업종**: "기타 정보 서비스업" (KSIC 63999) 또는 "응용 소프트웨어 개발 및 공급업" (KSIC 58222)
- 부가통신사업 신고로 충분 (게임물 유통업 등록 불필요)

#### 2.4.5 리스크: 게관위가 "게임물"로 판단할 경우

게관위는 실질적 내용을 기준으로 판단하므, 명칭만 바꾼다고 완전히 안전하지는 않다. 그러나:

1. **판타지 스포츠 선례**가 강력한 방어 논거 — 동일한 구조(실명 선수 + 의사결정 + 시뮬레이션 결과)로 정보 서비스 인정
2. **3D 그래픽/캐릭터 조작 없음** — 영상물 요건 미충족 (데이터/텍스트 기반)
3. **사행성 요소 없음** — 현금 보상, 확률형 아이템 없음
4. **최악의 경우**: 게관위 심의 요청 시 "전체이용가" 등급 취득 후 운영 가능 (비용은 발생하나 서비스 중단은 아님)

> **결론**: 포지셔닝만 올바르게 하면 게관위 규제를 회피할 수 있는 강력한 법적 근거가 있다. 다만 이는 명칭 변경만이 아닌, 서비스 전체의 프레이밍을 일관되게 "분석/시뮬레이션 플랫폼"으로 유지해야 한다.

## Part 3: 실명 유지 전략 (리스크 순)

### 전략 A: NBPA 라이선스 (가장 안전, 고비용)
- NBPA에 그룹 라이선스 신청 (매출의 10-15% 또는 연 $25K-50K 최소보장)
- 매출 $5K+/월 도달 후 협상 시작 권장

### 전략 B: 면책 강화 + 옵트아웃 체계 (실용적 중간)
- 이용약관에 명시: "공인의 성명을 정보/오락 목적으로 식별 차원에서 사용"
- NBPA/NBA 비제휴 명시, 보증 부인
- 선수/에이전트 요청 시 삭제 절차 마련 (이메일)
- **현 단계에서 즉시 적용 권장**

### 전략 C: 커뮤니티 패치 방식 (가장 보수적, 최소 리스크)
- 기본 제품은 가명 사용 (PES/Winning Eleven 방식)
- 커뮤니티가 실명 CSV를 별도 배포 → 유저가 수동 적용
- 상업 제품 자체에는 실명 없음 → 퍼블리시티권 회피
- **NBPA 라이선스 실패 시 폴백**

**권장:** 현재 → 전략 B 즉시 적용, 매출 성장 시 → 전략 A 협상, 실패 시 → 전략 C 전환

## Part 4: 수익화 모델 비교

### 4.1 구독형 (SaaS)
- **구조**: 무료 싱글플레이 / Pro ₩4,900/월 / League ₩9,900/월
- **수익 잠재력**: 1,000명 × ₩4,900 = ₩5.88M/월 (~$4.3K)
- **법적 리스크**: 중-높음 (반복 상업 매출 → NBPA 주목 가능성 ↑)
- **구현 난이도**: 높음 (결제 인프라 + 구독 관리)

### 4.2 일회성 구매 + DLC
- **구조**: 기본 ₩9,900-₩19,900 + DLC (멀티플레이어 ₩14,900, 멀티시즌 ₩9,900)
- **수익 잠재력**: 초기 매출 높으나 LTV 낮음
- **법적 리스크**: 중간
- **구현 난이도**: 중간 (결제 링크 + `isPurchased` boolean)

### 4.3 프리미엄 (Freemium)
- **구조**: 무료(싱글플레이 1시즌 1세이브) / 유료(멀티시즌, 무제한 세이브, 멀티, 통계 내보내기)
- **전환율**: MAU의 2-5% (10,000 MAU → 200-500명 유료)
- **법적 리스크**: 중간 (무료 기본 제공 → 선의 입증에 유리)
- **구현 난이도**: 중간 (Supabase RLS + 피처 게이팅)

### 4.4 광고 기반
- **수익 잠재력**: 매우 낮음 (니치 웹앱, 100K PV/월 → ₩50-150K/월)
- **법적 리스크**: 최저 (무료 서비스 → 퍼블리시티권 적용 약화)
- **구현 난이도**: 낮음

### 4.5 하이브리드 (권장)
- **Free**: 현재 싱글플레이 전체 기능
- **Pro** ₩3,900/월 (₩35,000/년): 멀티시즌 + 고급 분석 + 무제한 세이브
